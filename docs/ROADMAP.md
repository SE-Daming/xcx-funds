## 需求池

### 需求1：定投功能 ✅ 已设计完成

#### 用户诉求
设置定投计划后，系统自动模拟每日/每周/每月买入，无需手动记录每笔交易。

---

#### 功能概述

| 项目 | 说明 |
|------|------|
| 定投周期 | 每日 / 每周（可选周一到周五）/ 每月（可选1-28号）|
| 每期金额 | 用户自定义，单位：元 |
| 开始日期 | 限制在一年内（API限制）|
| 执行方式 | 用户手动点击"同步定投"按钮 |
| 份额精度 | 保留2位小数 |

---

#### 核心规则

**非交易日处理**：
- 每日定投：跳过该期，不补
- 每周/每月定投：顺延到下一交易日

**定投与持仓关系**：
- 定投份额合并到持仓 `num` 字段
- 定投成本合并计算，更新 `cost` 字段
- 支持与手动持仓并存（追加）

**计划管理**：
- 每个基金只能有一个定投计划
- 支持修改（从修改日起生效，之前的不变）
- 支持暂停/恢复
- 删除计划时，已定投份额保留

---

#### 数据结构

```javascript
// 基金数据扩展
{
  // ... 现有字段
  
  // 定投计划
  investPlan: {
    enabled: true,                          // 是否启用（false=暂停）
    cycle: 'daily' | 'weekly' | 'monthly',  // 定投周期
    amount: 10,                             // 每期金额（元）
    dayOfWeek: 1,                           // 每周几（1-5=周一到周五）
    dayOfMonth: 1,                          // 每月几号（1-28）
    startDate: '2024-01-01',                // 开始日期
    lastInvestDate: '2024-06-15'            // 上次执行日期
  },
  
  // 定投记录明细
  investRecords: [
    { date: '2024-01-02', amount: 10, nav: 1.025, shares: 9.76 },
    { date: '2024-01-03', amount: 10, nav: 1.018, shares: 9.82 }
  ]
}
```

---

#### 计算逻辑

```
输入：investPlan, 历史净值列表

1. 根据 cycle 和 startDate 生成定投日期列表
2. 过滤已执行的日期（<= lastInvestDate）
3. 处理非交易日：
   - 每日：跳过
   - 每周/每月：顺延到下一交易日
4. 对每个待执行日期：
   - 查找当日净值（确认净值，非预估）
   - 买入份额 = amount / nav
   - 份额保留2位小数
5. 汇总：
   - 累计份额 = Σ shares
   - 累计投入 = Σ amount
   - 定投均价 = 累计投入 / 累计份额
6. 更新持仓：
   - num += 累计份额
   - cost = 重新计算加权平均成本
```

#### 展示内容

**详情页 - 定投统计区块**：
- 累计投入金额
- 累计份额
- 当前市值（累计份额 × 当前净值）
- 定投收益率
- 定投均价
- 已执行期数
- 定投计划状态（进行中/已暂停）
- 定投记录明细（可展开查看）
- "同步定投"按钮

**编辑页 - 定投计划设置**：
- 启用/暂停开关
- 周期选择（每日/每周/每月）
- 金额输入
- 日期选择（周几/几号，根据周期显示）
- 开始日期选择（限制一年内）

**首页卡片**：
- 定投标记（如 📅 图标，可选）

#### API依赖

使用现有 `getFundHistoryNav(fundCode, range)` 获取历史净值：
- `y` = 近1月
- `3y` = 近3月
- `6y` = 近半年
- `n` = 近1年

**限制**：开始日期只能在一年内

### 需求2：加减仓功能（待设计）

**用户诉求**：支持记录基金的加仓、减仓操作，准确计算持仓成本

**方案A：交易记录模式**

在基金数据上增加交易流水：
```
transactions: [
  { type: 'buy', date: '2024-01-01', shares: 100, price: 1.0, amount: 100 },
  { type: 'sell', date: '2024-06-01', shares: 50, price: 1.2, amount: 60 },
  { type: 'buy', date: '2024-08-01', shares: 50, price: 0.9, amount: 45 }
]
```

展示内容：
- 持仓成本自动计算（加权平均）
- 累计投入、累计取出
- 分批建仓/减仓记录
- 每笔交易盈亏（针对已卖出部分）

优点：完整记录交易历史，成本计算准确
缺点：数据结构改动较大，需新增交易记录管理页面

---

**方案B：简单加减仓模式**

保持现有的 `num`（份额）和 `cost`（成本价）字段，新增：
```
adjustments: [
  { type: 'add', date: '2024-06-01', shares: 50, price: 0.95 },
  { type: 'reduce', date: '2024-08-01', shares: 30, price: 1.1 }
]
```

编辑页支持：
- 加仓：输入新增份额和买入价，自动更新总份额和成本价
- 减仓：输入卖出份额和卖出价，计算本次盈亏

优点：改动较小，符合用户习惯
缺点：历史记录有限，不够详细

---

**方案C：手动更新模式（当前已有）**

用户直接修改份额和成本价，无交易记录。

优点：最简单
缺点：无历史追溯，成本计算不透明
