> 更新日期：2026-04-17

## 一、功能概述

### 1.1 用户诉求

用户希望设置定投计划后，系统能自动模拟每日/每周/每月买入基金，无需手动记录每笔交易，自动计算累计投入、持仓份额、收益率等数据。

### 1.2 功能定位

- **自动模拟**：用户只需设置计划参数，系统自动计算每期买入
- **手动触发**：无后端服务，由用户手动点击"同步定投"执行
- **数据合并**：定投份额与手动持仓合并，统一计算成本和收益

### 1.3 功能概览

| 项目 | 说明 |
|------|------|
| 定投周期 | 每日 / 每周（可选周一到周五）/ 每月（可选1-28号）|
| 每期金额 | 用户自定义，单位：元 |
| 开始日期 | 限制在一年内（受API限制）|
| 执行方式 | 用户手动点击"同步定投"按钮 |
| 份额精度 | 记录保留4位小数，显示时四舍五入到2位 |

## 二、核心规则

### 2.1 定投周期规则

**每日定投**：
- 每个交易日自动买入
- 非交易日（周末、节假日）跳过该期，不补

**每周定投**：
- 用户选择周一到周五某一天
- 若该日为非交易日，顺延到下一交易日

**每月定投**：
- 用户选择1-28号某一天
- 若该日为非交易日，顺延到下一交易日
- 注：限制1-28号是为了避免部分月份没有29/30/31日的问题

### 2.2 非交易日处理

```
每日定投：
  定投日 = 非交易日 → 跳过，不补

每周/每月定投：
  定投日 = 非交易日 → 顺延到下一交易日
  若下一交易日已跨月（每月定投），则跳过本月
```

### 2.3 定投与持仓关系

| 场景 | 处理方式 |
|------|----------|
| 已有手动持仓，开启定投 | 定投份额追加到现有持仓 |
| 只有定投，无手动持仓 | 持仓完全由定投累积 |
| 暂停定投 | 已累积份额保留在持仓中 |
| 终止定投 | 保留份额和记录，计划标记为已终止 |
| 删除定投记录 | 可选：仅删除计划 / 同时清空记录和份额 |

**成本计算**：
- 定投会产生新的份额和成本
- 需要与现有持仓合并计算加权平均成本
- 成本价 = 总投入金额 / 总份额

### 2.4 净值获取规则

- 使用**确认净值**（收盘后公布的净值），非预估净值
- 同步定投时，只能同步到最近一个有确认净值的交易日
- 当天交易日的净值需等收盘后才能获取

### 2.5 修改计划规则

| 修改项 | 影响 | 说明 |
|--------|------|------|
| 修改金额 | 从下一期生效 | 已执行记录不变，后续按新金额计算 |
| 修改周期 | 从下一期生效 | 需重新判断定投日期，已执行记录不变 |
| 修改定投日（周/月） | 从下一期生效 | 已执行记录不变 |
| 修改开始日期（提前） | 不可操作 | 日期选择器限制，只能选今天或未来 |
| 修改开始日期（延后） | 可操作 | 相当于重新设置计划起点 |

**重要提示**：修改计划参数后，应在保存时提示用户「修改将从下一期生效，已执行记录不变」。

### 3.2 字段说明

**investPlan 字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| enabled | Boolean | 是 | 是否启用定投计划 |
| status | String | 是 | 计划状态：active=进行中，paused=已暂停，terminated=已终止 |
| cycle | String | 是 | 定投周期：daily/weekly/monthly |
| amount | Number | 是 | 每期投入金额，单位：元 |
| dayOfWeek | Number | 否 | 每周几定投（1-5=周一到周五），cycle=weekly 时必填 |
| dayOfMonth | Number | 否 | 每月几号定投（1-28），cycle=monthly 时必填 |
| startDate | String | 是 | 定投开始日期，格式：YYYY-MM-DD |
| lastInvestDate | String | 否 | 上次执行定投的日期，用于增量同步 |
| terminatedDate | String | 否 | 终止日期，status=terminated 时记录 |

**investRecords 字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| date | String | 定投执行日期 |
| amount | Number | 投入金额 |
| nav | Number | 当日确认净值 |
| shares | Number | 买入份额（保留4位小数，显示时四舍五入到2位）|

### 3.3 份额精度说明

为避免长期累积误差：
- **存储精度**：每条记录的 `shares` 保留4位小数
- **显示精度**：界面显示时四舍五入到2位小数
- **计算精度**：汇总计算时使用4位小数，最终结果四舍五入

示例：
```
单期：10元 / 1.025 = 9.7561份（存4位）
100期后：累计份额 = 975.61份（直接求和，无精度损失）
显示：975.61份
```

## 四、计算逻辑

### 4.1 定投日期生成算法

```javascript
/**
 * 生成定投日期列表
 * @param {Object} plan - 定投计划
 * @param {String} endDate - 截止日期（通常为今天或最近交易日）
 * @param {Array} tradingDays - 交易日列表（从历史净值数据提取）
 * @returns {Array} 定投日期列表
 */
function generateInvestDates(plan, endDate, tradingDays) {
  const dates = [];
  const startDate = new Date(plan.startDate);
  const end = new Date(endDate);

  // 获取交易日集合（用于判断非交易日）
  const tradingDaySet = new Set(tradingDays);

  // 根据周期类型生成候选日期
  switch (plan.cycle) {
    case 'daily':
      // 每日定投：直接使用交易日列表
      for (const day of tradingDays) {
        if (day >= plan.startDate && day <= endDate && day > (plan.lastInvestDate || '')) {
          dates.push(day);
        }
      }
      break;

    case 'weekly':
      // 每周定投：找到每周指定日
      dates.push(...generateWeeklyDates(plan, startDate, end, tradingDaySet));
      break;

    case 'monthly':
      // 每月定投：找到每月指定日
      dates.push(...generateMonthlyDates(plan, startDate, end, tradingDaySet));
      break;
  }

  return dates;
}

/**
 * 生成每周定投日期
 * @param {Object} plan - 定投计划
 * @param {Date} startDate - 开始日期
 * @param {Date} endDate - 截止日期
 * @param {Set} tradingDaySet - 交易日集合
 */
function generateWeeklyDates(plan, startDate, endDate, tradingDaySet) {
  const dates = [];
  const targetDay = plan.dayOfWeek; // 1-5 表示周一到周五

  // 周几对应的 JS getDay() 值
  // 1=周一 → 1, 2=周二 → 2, ..., 5=周五 → 5
  const jsDay = targetDay; // 直接对应

  let current = new Date(startDate);

  // 找到第一个目标周几
  while (current.getDay() !== jsDay) {
    current.setDate(current.getDate() + 1);
  }

  // 遍历每个目标周几
  while (current <= endDate) {
    const dateStr = formatDate(current);

    if (dateStr > (plan.lastInvestDate || '')) {
      // 检查是否为交易日，非交易日顺延
      if (tradingDaySet.has(dateStr)) {
        dates.push(dateStr);
      } else {
        // 顺延到下一个交易日
        const nextTradingDay = findNextTradingDay(dateStr, tradingDaySet);
        if (nextTradingDay && nextTradingDay <= formatDate(endDate)) {
          dates.push(nextTradingDay);
        }
      }
    }

    // 移动到下周同一天
    current.setDate(current.getDate() + 7);
  }

  return dates;
}

/**
 * 生成每月定投日期
 * @param {Object} plan - 定投计划
 * @param {Date} startDate - 开始日期
 * @param {Date} endDate - 截止日期
 * @param {Set} tradingDaySet - 交易日集合
 */
function generateMonthlyDates(plan, startDate, endDate, tradingDaySet) {
  const dates = [];
  const targetDay = plan.dayOfMonth; // 1-28

  let year = startDate.getFullYear();
  let month = startDate.getMonth();

  while (true) {
    // 创建当月目标日期（使用 new Date 避免溢出问题）
    const targetDate = new Date(year, month, targetDay);

    // 检查是否超出范围
    if (targetDate > endDate) break;

    const dateStr = formatDate(targetDate);

    if (dateStr >= plan.startDate && dateStr > (plan.lastInvestDate || '')) {
      // 检查是否为交易日
      if (tradingDaySet.has(dateStr)) {
        dates.push(dateStr);
      } else {
        // 顺延到下一个交易日
        const nextTradingDay = findNextTradingDay(dateStr, tradingDaySet);
        // 检查是否跨月
        if (nextTradingDay) {
          const nextDate = new Date(nextTradingDay);
          if (nextDate.getMonth() === month && nextDate <= endDate) {
            dates.push(nextTradingDay);
          }
          // 跨月则跳过本月
        }
      }
    }

    // 移动到下个月
    month++;
    if (month > 11) {
      month = 0;
      year++;
    }
  }

  return dates;
}

/**
 * 查找下一个交易日（顺延）
 * @param {String} dateStr - 起始日期字符串
 * @param {Set} tradingDaySet - 交易日集合
 * @returns {String|null} 下一个交易日，找不到返回null
 */
function findNextTradingDay(dateStr, tradingDaySet) {
  let date = new Date(dateStr);
  let attempts = 0;
  const maxAttempts = 10; // 最多顺延10天

  while (attempts < maxAttempts) {
    date.setDate(date.getDate() + 1);
    const nextDateStr = formatDate(date);
    if (tradingDaySet.has(nextDateStr)) {
      return nextDateStr;
    }
    attempts++;
  }

  return null; // 找不到交易日
}

/**
 * 格式化日期为 YYYY-MM-DD
 */
function formatDate(date) {
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, '0');
  const day = String(date.getDate()).padStart(2, '0');
  return `${year}-${month}-${day}`;
}
```

### 4.2 份额计算与合并

```javascript
/**
 * 执行定投计算
 * @param {Object} plan - 定投计划
 * @param {Array} navHistory - 历史净值列表 [{date, nav}, ...]
 * @param {Array} existingRecords - 已有定投记录
 * @returns {Object} 计算结果
 */
function executeInvestment(plan, navHistory, existingRecords) {
  // 提取交易日列表
  const tradingDays = navHistory.map(item => item.date);
  const navMap = new Map(navHistory.map(item => [item.date, item.nav]));

  // 获取最近有净值的交易日
  const lastTradingDay = tradingDays[tradingDays.length - 1];

  // 生成待执行定投日期
  const investDates = generateInvestDates(plan, lastTradingDay, tradingDays);

  // 计算每期份额
  const newRecords = [];
  let totalShares = 0;
  let totalAmount = 0;

  for (const date of investDates) {
    const nav = navMap.get(date);
    if (!nav) continue;

    // 保留4位小数存储
    const shares = parseFloat((plan.amount / nav).toFixed(4));

    newRecords.push({
      date,
      amount: plan.amount,
      nav,
      shares
    });

    totalShares += shares;
    totalAmount += plan.amount;
  }

  // 合并已有记录
  const allRecords = [...existingRecords, ...newRecords];

  // 计算汇总数据
  const summary = calculateSummary(allRecords, navMap.get(lastTradingDay));

  return {
    newRecords,
    allRecords,
    summary,
    lastInvestDate: investDates.length > 0 ? investDates[investDates.length - 1] : plan.lastInvestDate
  };
}

/**
 * 计算定投汇总数据
 */
function calculateSummary(records, currentNav) {
  const totalAmount = records.reduce((sum, r) => sum + r.amount, 0);
  const totalShares = records.reduce((sum, r) => sum + r.shares, 0);
  const currentValue = totalShares * currentNav;
  const avgCost = totalShares > 0 ? totalAmount / totalShares : 0;
  const profit = currentValue - totalAmount;
  const profitRate = totalAmount > 0 ? (profit / totalAmount * 100).toFixed(2) : '0.00';

  return {
    totalAmount,       // 累计投入
    totalShares,       // 累计份额（保留原始精度用于计算）
    currentValue,      // 当前市值
    avgCost,           // 定投均价
    profit,            // 累计收益
    profitRate,        // 收益率
    investCount: records.length  // 已执行期数
  };
}
```

### 4.3 持仓成本合并

```javascript
/**
 * 合并定投份额到持仓
 * @param {Object} fund - 基金数据
 * @param {Number} newShares - 新增份额
 * @param {Number} newAmount - 新增投入
 */
function mergeToHolding(fund, newShares, newAmount) {
  // 计算原有持仓成本（考虑原有持仓可能为空）
  const oldNum = parseFloat(fund.num) || 0;
  const oldCost = parseFloat(fund.cost) || 0;
  const oldAmount = oldNum * oldCost;

  // 合并计算
  const totalShares = oldNum + newShares;
  const totalAmount = oldAmount + newAmount;

  if (totalShares > 0) {
    // 更新持仓（份额保留2位，成本保留4位）
    fund.num = parseFloat(totalShares.toFixed(2));
    fund.cost = parseFloat((totalAmount / totalShares).toFixed(4));
  }
}
```

## 九、风险与限制

### 9.1 已知限制

1. **时间范围限制**：开始日期只能在一年内
2. **手动触发**：需要用户手动点击同步，无法自动执行
3. **净值延迟**：当天净值需等收盘后才能获取

### 9.2 潜在风险

1. **数据量**：长期定投会产生大量记录
   - 解决方案：记录列表分页加载，支持清空历史记录
2. **净值缺失**：部分日期可能没有净值数据
   - 解决方案：遇到缺失日期跳过，记录日志供用户查看
3. **数据迁移**：新增字段需考虑旧数据兼容性
   - 解决方案：加载数据时检查字段是否存在，不存在则初始化默认值

## 十、后续优化方向

1. **自动同步**：支持后台自动执行定投（需后端支持）
2. **定投提醒**：推送通知提醒用户同步定投
3. **定投分析**：定投收益曲线图、与一次性买入对比
4. **智能定投**：根据市场情况动态调整金额（如低估多投）
5. **记录导出**：支持导出定投记录为 Excel/CSV
